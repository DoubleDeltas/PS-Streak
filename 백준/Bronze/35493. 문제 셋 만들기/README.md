### 문제 설명

<p>2026 KPSC Spring Algorithm Challenge를 준비 중인 찬영이는 대회 문제를 어떻게 구성할지 고민 중이다. 짝수를 너무 좋아하는 찬영이는 문제 난이도의 총합이 짝수인 셋을 좋은 셋이라고 생각한다.</p>

<p>현재 대회는 $N$개의 문제가 준비되어 있다. 찬영이는 출제자들의 노고를 반영하기 위해 최대 $1$문제까지만 제거하되, 적어도 $1$문제는 남기면서 난이도의 총합은 짝수가 되게 만들려고 한다.</p>

<p>찬영이가 좋은 셋을 만들어 만족할 수 있는 대회를 열 수 있을지 알려주자.</p>

### 입력 

 <p>첫 번째 줄에 문제의 개수 $N$이 주어진다.</p>

<p>두 번째 줄에 $i$번째 문제의 난이도를 나타내는 정수 $D_1, D_2, \cdots, D_N$이 공백으로 구분되어 주어진다.</p>

### 출력 

 <p>첫 번째 줄에 찬영이가 좋은 셋을 만들 수 있으면 <code><span style="color:#e74c3c;">YES</span></code>를, 아니면 <code><span style="color:#e74c3c;">NO</span></code>를 출력한다.</p>
